# DANZK Workspace

## 📋 Descripción General

**DANZK Workspace** es un espacio de desarrollo colaborativo diseñado para la creación y gestión de proyectos web modernos. Este repositorio centraliza los componentes, herramientas y recursos necesarios para desarrollar soluciones digitales de alta calidad utilizando tecnologías actuales.

## 🎯 Propósito del Proyecto

Este proyecto sirve como:

- **Centro de desarrollo**: Plataforma centralizada para construir y mantener aplicaciones web
- **Espacio colaborativo**: Entorno estructurado para el trabajo en equipo de desarrolladores
- **Base de conocimiento**: Repositorio de mejores prácticas, componentes reutilizables y configuraciones optimizadas
- **Ecosistema integrado**: Conjunto de herramientas y servicios interconectados para acelerar el desarrollo

## 🚀 Características Principales

- **Stack Moderno**: Utiliza React + Vite para desarrollo rápido y eficiente
- **Landing Page Profesional**: Sitio web de presentación completamente funcional
- **Configuración Optimizada**: ESLint y herramientas de calidad de código preconfiguradas
- **HMR (Hot Module Replacement)**: Recarga en caliente para experiencia de desarrollo mejorada
- **Escalabilidad**: Estructura preparada para crecer y adaptarse a nuevos requisitos

## 📁 Estructura del Proyecto

```
DANZK-WORKSPACE/

├── README.md              # Este archivo

```

### Danzk Landing Page

Landing page responsiva desarrollada con:
- **React** - Biblioteca para construir interfaces de usuario
- **Vite** - Bundler y servidor de desarrollo ultrarrápido
- **ESLint** - Linting y análisis estático de código

## 🛠️ Requisitos Previos

- Node.js (versión 16 o superior)
- npm o yarn
- Git

## ⚙️ Instalación y Setup

1. **Clonar el repositorio**
   ```bash
   git clone https://github.com/danzkpalomino-ai/DANZK-WORKSPACE.git
   cd DANZK-WORKSPACE
   ```

2. **Instalar dependencias**
   ```bash
   cd "Danzk Landing Page"
   npm install
   ```

3. **Ejecutar el servidor de desarrollo**
   ```bash
   npm run dev
   ```

4. **Construir para producción**
   ```bash
   npm run build
   ```

## 📝 Guía para Colaboradores

### Antes de Contribuir

1. Fork del repositorio
2. Crear una rama feature (`git checkout -b feature/tu-caracteristica`)
3. Hacer commits descriptivos (`git commit -m 'Add: descripción clara'`)
4. Push a tu rama (`git push origin feature/tu-caracteristica`)
5. Abrir un Pull Request

### Estándares de Código

- Seguir las reglas de ESLint configuradas
- Escribir código limpio y legible
- Documentar cambios significativos
- Realizar pruebas locales antes de hacer push
- Usar conventional commits para mensajes

### Plugins Disponibles

#### Opción 1: Oxc
```bash
npm install -D @vitejs/plugin-react
```

#### Opción 2: SWC (Alternativa de mayor rendimiento)
```bash
npm install -D @vitejs/plugin-react-swc
```

## 🔧 Configuración Avanzada

### React Compiler

El compilador de React está deshabilitado por defecto para optimizar el rendimiento en desarrollo. Para habilitarlo, consulta la [documentación oficial](https://react.dev/learn/react-compiler/installation).

### ESLint Mejorado

Para una aplicación en producción con TypeScript, se recomienda configurar reglas de lint conscientes de tipos. Consulta la [plantilla de Vite + TypeScript](https://github.com/vitejs).
